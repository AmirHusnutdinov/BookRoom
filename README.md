# BookRoom

Система бронирования переговорных комнат и аудиторий. Веб-приложение на Flask с авторизацией, календарём занятости, личным кабинетом пользователей и админ-панелью. Контейнеризировано с Docker для быстрого и воспроизводимого деплоя.

## Возможности

-  Регистрация и вход с разграничением ролей (`пользователь` / `админ`)
-  Каталог комнат с характеристиками (вместимость, этаж, оборудование)
-  Календарь занятости с фильтрацией по дате и комнате
-  Бронирование времени с автоматической проверкой конфликтов
-  Личный кабинет: предстоящие и прошедшие бронирования
-  Отмена бронирований (только пользователем-владельцем)
-  Админ-панель: добавление/активация комнат, управление пользователями, просмотр всех броней

##  Стек технологий

| Компонент       | Технология                            |
|-----------------|---------------------------------------|
| **Backend**     | Python 3.10+, Flask 3.x               |
| **Frontend**    | HTML5, CSS3, Vanilla JavaScript       |
| **Database**    | SQLite 3                              |
| **Auth**        | Werkzeug (password hashing, sessions) |
| **Deployment**  | Docker, Docker Compose, Gunicorn      |

### Требования
- Docker ≥ 20.10
- Docker Compose ≥ 1.29 (или плагин `docker compose` v2)

### Установка и запуск

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/AmirHusnutdinov/BookRoom.git
cd BookRoom
ls -la
```

2. **Настройте переменные окружения:**
```bash
cp .env.example .env
```
Откройте `.env` и укажите свои значения.

3. **Запустите приложение:**
```bash
# Для Docker Compose v2:
docker compose up -d --build

# Для Docker Compose v1 (старая версия):
docker-compose up -d --build
```

4. **Откройте в браузере:**
 `http://localhost:5000`

## Конфигурация

Переменные задаются в файле `.env`:

| Переменная      | Описание                                      | Пример                                      |
|-----------------|-----------------------------------------------|---------------------------------------------|
| `SECRET_KEY`    | Криптографический ключ сессий Flask           | `python3 -c "import secrets; print(secrets.token_hex(32))"` |
| `ADMIN_SECRET`  | Код для получения роли админа при регистрации | `MyTopSecret2025!`                          |
| `FLASK_APP`     | Точка входа приложения                        | `app.py`                                    |
| `FLASK_DEBUG`   | Режим отладки (`1` для dev, `0` для prod)     | `0`                                         |

##  Деплой на сервер (VPS)

1. Загрузите проект на сервер (через `git clone` или SFTP)
2. Создайте `.env` с уникальными секретами и `FLASK_DEBUG=0`
3. Запустите: `docker compose up -d --build`
4. Настройте обратный прокси (Nginx/Caddy) и HTTPS (Let's Encrypt) для продакшена

>  Для production рекомендуется вынести SQLite в PostgreSQL/MySQL и использовать Gunicorn с `workers > 1`. В текущей конфигурации Gunicorn запускается автоматически.

##  Безопасность и лучшие практики

-  `SECRET_KEY` должен быть случайным и уникальным для каждого окружения
-  `FLASK_DEBUG` всегда `0` на публичных серверах
-  `.env` и `*.db` добавлены в `.gitignore` — не попадут в Git
-  Пароли хешируются через `werkzeug.security.generate_password_hash`
-  Сессии подписываются криптографически и не могут быть подделаны без `SECRET_KEY`

##  Устранение неполадок

| Проблема                          | Решение                                                                 |
|-----------------------------------|-------------------------------------------------------------------------|
| `Container exited with code 1`    | `docker compose logs --tail=50` — посмотрите причину ошибки в логах   |
| `ModuleNotFoundError: flask`      | `docker compose build --no-cache && docker compose up -d`               |
| Сайт недоступен по `localhost:5000` | Проверьте `docker compose ps`. Убедитесь, что порт `5000` проброшен    |
| `sqlite3.OperationalError`        | Убедитесь, что контейнер имеет права на запись в папку с `booking.db`   |
